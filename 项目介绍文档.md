## 一、项目背景与目标

### 1.1 项目愿景

**让全球闲置算力不再浪费**

Idle-Sense 是一个分布式闲置算力共享平台，致力于释放全球个人电脑闲置算力的巨大潜能。我们相信，世界上有数以亿计的个人电脑在大部分时间里处于闲置状态，这些闲置的算力是一种巨大的、被忽视的资源。

### 1.2 问题陈述

| 问题类型 | 具体描述 | 市场影响 |
|---------|---------|---------|
| 算力浪费 | 全球个人电脑平均利用率不足 20% | 数十亿计的算力资源被浪费 |
| 云计算成本高 | 传统云计算服务价格昂贵 | 中小企业和个人开发者负担沉重 |
| 中心化风险 | 过度依赖中心化云服务商 | 数据隐私、服务中断、价格垄断风险 |

### 1.3 项目目标

#### 短期目标（已实现）
- ✅ 构建跨平台闲置检测系统（Windows/macOS/Linux）

- ✅ 建立代币经济激励机制
- ✅ 提供 Web 管理界面

#### 中期目标（进行中）

- 🔄 吸引首批外部贡献者
- 🔄 发布 v1.0.0 正式版

#### 长期目标
- 📋 实现完全去中心化的算力共享网络
- 📋 建立联邦协议支持跨组织协作
- 📋 构建可持续发展的代币经济生态

### 2.2 智能闲置检测

#### 检测维度

| 检测维度 | 阈值 | 说明 | 优先级 |
|---------|------|------|--------|
| 用户活动 | >300秒 | 无键盘/鼠标操作 | 必须 |
| CPU使用率 | <30% | 系统负载较低 | 必须 |
| 内存使用率 | <70% | 内存充足 | 必须 |
| 屏幕状态 | 锁定 | 用户不在场 | 可选 |
| 电源状态 | 充电中 | 不消耗电池 | 可选 |

#### 检测算法

```python
def is_truly_idle() -> bool:
    """
    五重验证的闲置判定算法
    全部条件满足才返回 True
    """
    return all([
        idle_time > 300,           # 条件1: 5分钟无操作
        cpu_usage < 30.0,          # 条件2: CPU空闲70%以上
        memory_usage < 70.0,       # 条件3: 内存充足
        screen_locked,             # 条件4: 屏幕锁定
        is_charging or not has_battery  # 条件5: 不消耗用户电量
    ])
```

### 2.4 公平调度算法

#### 调度策略参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 等待时间权重 | 60% | 主要考虑因素 |
| 贡献度权重 | 30% | 适度奖励贡献 |
| 新人加成权重 | 10% | 保证新节点机会 |
| 贡献度上限 | 10分 | 防止过度倾斜 |
| 防饥饿阈值 | 300秒 | 5分钟自动最高优先级 |
| 冷却时间 | 1800秒 | 30分钟内降低优先级 |

#### 优先级计算公式

```
节点优先级 = 等待时间得分(60%) + 贡献度得分(30%) + 新人加成(10%)

其中:
- 等待时间得分 = -wait_time_seconds × 0.6
- 贡献度得分 = -min(tasks_completed × 0.1, 10.0) × 0.3
- 新人加成 = -(20 - tasks_completed × 2) × 0.1 (仅前10个任务)
```

### 3.4 核心实体模型

#### Task 实体

```python
@dataclass
class Task:
    task_id: str                    # 唯一标识
    code: str                       # 执行代码
    status: TaskStatus              # 状态（pending/running/completed/failed/cancelled）
    user_id: Optional[str]          # 所属用户
    timeout: int = 300              # 超时时间
    cpu_request: float = 1.0        # CPU 请求
    memory_request: int = 512       # 内存请求（MB）
    task_type: TaskType             # 任务类型
    assigned_node: Optional[str]    # 分配节点
    result: Optional[str]           # 执行结果
    error: Optional[str]            # 错误信息
```

#### Node 实体

```python
@dataclass
class Node:
    node_id: str                    # 唯一标识
    platform: str                   # 平台（Windows/macOS/Linux）
    status: NodeStatus              # 状态（online/offline/busy/idle）
    capacity: dict[str, Any]        # 资源容量
    tags: dict[str, str]            # 标签
    owner: str                      # 所有者
    last_heartbeat: Optional[datetime]  # 最后心跳时间
    is_available: bool              # 是否可用
    is_idle: bool                   # 是否空闲
```

### 7.1 快速开始

#### 安装

```bash
# 克隆仓库
git clone https://github.com/Xing-Heyu/idle-sense.git
cd idle-sense

# 安装依赖
pip install -r requirements.txt
```

#### 启动服务

```bash
# 启动调度中心
python -m legacy.scheduler.simple_server

# 启动计算节点
python -m legacy.node.simple_client --scheduler http://localhost:8000

# 启动 Web 界面
streamlit run src/presentation/streamlit/app.py
```
