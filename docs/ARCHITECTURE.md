# 系统架构文档

## 概述

Idle-Sense 是一个分布式闲置算力共享平台，采用分层架构设计，确保系统的可维护性、可扩展性和安全性。

## 核心组件

### 1. 调度中心 (Scheduler)

调度中心是系统的核心组件，负责：

- **任务管理**: 接收、排队、分发任务
- **节点管理**: 注册、心跳检测、状态跟踪
- **调度算法**: 公平优先调度，防止饥饿
- **结果存储**: 存储任务执行结果

### 2. 计算节点 (Node)

计算节点是执行任务的客户端：

- **闲置检测**: 多维度检测电脑闲置状态
- **任务执行**: 在沙箱环境中安全执行代码
- **结果上报**: 将执行结果返回调度中心
- **心跳维持**: 定期发送心跳保持在线

### 3. 闲置检测 (Idle Detection)

多维度闲置检测系统：

| 检测维度 | 阈值 | 说明 |
|---------|------|------|
| 用户活动 | 300秒 | 无键盘/鼠标操作 |
| CPU使用率 | <30% | 系统负载较低 |
| 内存使用率 | <70% | 内存充足 |
| 屏幕状态 | 锁定 | 用户不在场 |
| 电源状态 | 充电中 | 不消耗电池 |

## 数据流

### 任务提交流程

```
用户 → Web界面/CLI → API → 调度中心 → 任务队列
                                           ↓
结果 ← Web界面/CLI ← API ← 调度中心 ← 节点执行
```

### 节点调度流程

```
1. 节点检测到闲置状态
2. 节点发送心跳到调度中心
3. 调度中心选择任务分配
4. 节点接收任务
5. 节点在沙箱中执行
6. 节点上报结果
7. 调度中心存储结果
```

## 扩展点

### 1. 调度算法

可通过实现 `SchedulingAlgorithm` 接口添加新的调度策略：

```python
class SchedulingAlgorithm(ABC):
    @abstractmethod
    def select_node(self, task: Task, nodes: list[Node]) -> Node | None:
        pass
```

### 2. 沙箱实现

可通过实现 `Sandbox` 接口添加新的隔离机制：

```python
class Sandbox(ABC):
    @abstractmethod
    def execute(self, code: str, timeout: int) -> ExecutionResult:
        pass
```

### 3. 存储后端

可通过实现 `Repository` 接口替换存储方式：

```python
class TaskRepository(ABC):
    @abstractmethod
    def save(self, task: Task) -> None:
        pass
    
    @abstractmethod
    def find_by_id(self, task_id: str) -> Task | None:
        pass
```

## 安全考虑

### 1. 代码执行安全

- 所有代码在沙箱中执行
- 资源限制（CPU、内存、时间）
- 网络访问默认禁止
- 文件系统隔离

### 2. 通信安全

- HTTPS 加密传输
- 认证令牌机制
- 输入验证和清理

### 3. 数据安全

- 敏感信息不记录日志
- 任务结果定期清理
- 用户数据隔离

## 性能优化

### 1. 任务调度

- 内存队列快速响应
- 批量心跳处理
- 异步任务分发

### 2. 节点通信

- 长连接减少开销
- 心跳合并发送
- 结果压缩传输

### 3. 资源管理

- 连接池复用
- 对象池管理
- 延迟加载
